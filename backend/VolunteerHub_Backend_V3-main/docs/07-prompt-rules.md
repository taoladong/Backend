# Prompt Rules

Every code generation task must begin by reading:
- project overview
- business rules
- architecture
- coding standards
- module checklist
- progress
- target module file

Every response must include:
- scope summary
- assumptions
- file plan
- code file-by-file
- integration notes
- TODOs
- progress update suggestion

Never:
- rewrite unrelated modules
- break architecture
- hide assumptions
- omit validation without explanation
- omit tests without explanation
